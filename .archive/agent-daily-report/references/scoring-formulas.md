# Scoring Formulas Reference

## 唯一评分公式

总分 0-100 = relevance(30) + popularity(20) + freshness(15) + growth(15) + utility(20)

**不使用归一化**：各维度直接加权求和，每个 item 独立计算，不受其他 item 影响。

## 等级阈值

| 等级 | 阈值 | 含义 |
|------|------|------|
| S | ≥ 85 | 必须推送 |
| A | ≥ 70 | 重点关注 |
| B | ≥ 55 | 普通收录 |
| C | ≥ 40 | 备选 |
| D | < 40 | 忽略 |

## Growth 评分 (0-15)

`score_growth()` 优先级：
1. **日均增长**（从快照历史计算，2026-06-05 新增）
2. **单日增长**（star_delta_24h）
3. **实时飙升**（recent_star_window）
4. **星星数量**（最后手段）

### 日均增长阈值

| 日均增长 (stars/day) | 分数 |
|---------------------|------|
| > 500 | 15 |
| > 200 | 12 |
| > 100 | 9 |
| > 50 | 6 |
| > 20 | 3 |

### 日均增长保底B级

当日均增长 ≥ 20 stars/day 时，总分被提升到至少 55（B级下限）。
通过 `daily_avg_growth_boost` quality_flag 标记。

### 单日增长阈值（回退）

| 单日增长 (stars) | 分数 |
|-----------------|------|
| > 1000 | 15 |
| > 500 | 12 |
| > 200 | 9 |
| > 100 | 6 |
| > 50 | 3 |

## Trust Decision 对等级的影响

Trust Agent 可以对项目进行降级或升级，导致最终等级偏离评分对应的等级。

详见 `trust-decision-mechanism.md`
