# 分类审查执行清单

> **执行时机**：Step 4（分类）完成后，Step 5（生成报告）之前
> **目的**：验证分类结果的准确性，防止分类错误

## 审查项目

### 1. 分类一致性检查
- [ ] 检查是否有同一个项目出现在多个板块
- [ ] 检查是否有重复分类的项目
- [ ] 验证已分类项目字典是否完整

### 2. 分类准确性检查
- [ ] 逐项检查每个项目的分类标签
- [ ] 验证分类是否符合P24规则（MCP > Agent组件 > Agent > Skills）
- [ ] 检查是否有误判案例（参考P2.8错误案例）

### 3. 分类完整性检查
- [ ] 检查是否遗漏了重要项目
- [ ] 验证搜索结果是否都被分类
- [ ] 检查是否有项目被错误过滤

### 4. 分类标签验证
- [ ] Agent项目：是否可独立运行+有API/Web UI
- [ ] Skills项目：是否为.md规则文档
- [ ] Agent组件：是否为跨平台优化系统
- [ ] MCP项目：是否为Model Context Protocol相关

## 审查流程

### Step 1: 读取分类结果
```python
# 读取已分类项目字典
classified_projects = load_classified_projects()
```

### Step 2: 逐项检查分类标签
```python
for project_name, info in classified_projects.items():
    category = info['category']
    # 检查分类是否正确
    if not verify_classification(project_name, category):
        mark_for_correction(project_name)
```

### Step 3: 验证分类是否符合规则
```python
# 检查优先级：MCP > Agent组件 > Agent > Skills
for project_name, info in classified_projects.items():
    if info['category'] == 'mcp':
        # MCP项目不应该同时是Agent组件
        pass
    elif info['category'] == 'component':
        # Agent组件不应该同时是Agent
        pass
```

### Step 4: 标记并修正错误
```python
# 修正错误分类
for project_name in corrections:
    old_category = classified_projects[project_name]['category']
    new_category = correct_classification(project_name)
    classified_projects[project_name]['category'] = new_category
    print(f"修正: {project_name} {old_category} -> {new_category}")
```

## 常见错误案例

### 错误1: Agent误判为Skills
- **案例**: ARIS（描述含"Markdown-only skills"）
- **正确分类**: 📚 Skills
- **错误原因**: 看到Python代码就归为Agent

### 错误2: Agent组件误判为Agent
- **案例**: mcp-context-forge（AI Gateway）
- **正确分类**: 🧩 Agent组件
- **错误原因**: 看到可独立运行就归为Agent

### 错误3: MCP项目遗漏
- **案例**: github-mcp-server
- **正确分类**: 🔌 MCP
- **错误原因**: 没有检查MCP相关描述

## 注意事项

1. **分类审查必须在生成报告前执行**
2. **修正项必须记录修正原因**
3. **审查报告必须保存到日志**
4. **分类一致性是最高优先级**
