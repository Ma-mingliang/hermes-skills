## 🔍 MCP动态报告生成清单

### 1. MCP监控标准（必须确认纳入标准）
**目标**：确定哪些MCP项目值得纳入日报

**监控标准**（满足任一项即纳入）：
- [ ] Stars > 100 的新MCP服务器
- [ ] Stars > 5000 的MCP服务器有重大更新
- [ ] MCP协议本身有重要变更（版本、规范、安全公告）
- [ ] MCP生态重要事件（新SDK语言支持、重大收购、安全漏洞）
- [ ] 多源交叉确认的热点MCP项目（≥3源）

**排除标准**：
- [ ] Stars < 100 且无特殊意义的新项目（暂不纳入）
- [ ] 纯个人实验性MCP（无人关注）
- [ ] 重复性封装的MCP（如N个同类API封装）

**输出**：
```
MCP监控标准检查：
- 符合纳入标准：X个
- 排除（低星）：Y个
- 排除（重复）：Z个
```

### 3. MCP分类（必须）
**目标**：将MCP项目归入7大类别

**MCP七大分类及定义**：

| 分类 | 定义 | 典型示例 |
|------|------|---------|
| 🌐 浏览器控制 | 控制浏览器、网页交互、爬虫 | Playwright MCP、Puppeteer MCP |
| 💻 代码智能 | 代码分析、LSP、代码搜索、重构 | GitHub MCP、Sourcegraph MCP |
| 🗄️ 数据库 | 数据库连接、查询、Schema管理 | PostgreSQL MCP、SQLite MCP |
| 🔗 工作流自动化 | CI/CD、任务编排、流程自动化 | n8n MCP、Airflow MCP |
| 🔌 API集成 | 第三方API封装（搜索/邮件/日历/文件等） | Exa MCP、Slack MCP、Notion MCP |
| 🔒 安全 | 安全扫描、漏洞检测、权限管理 | Security MCP、1Password MCP |
| 🏗️ 开发框架 | MCP开发SDK、脚手架、测试工具 | FastMCP、MCP Builder |

**验证清单**：
- [ ] 每个MCP项目是否已分类
- [ ] 分类是否准确（对照定义检查）

### 4. MCP项目评估方法（必须）
**目标**：对MCP项目进行多维度评估

**评估维度**（每个项目至少评估3个维度）：

| 维度 | 评估标准 | 权重 |
|------|---------|------|
| 🌟 社区热度 | Stars数、近期增长趋势、讨论活跃度 | 高 |
| 🔧 功能完整度 | 覆盖的MCP能力范围、工具数量 | 高 |
| 📦 集成便利性 | 安装难度、配置复杂度、文档质量 | 中 |
| 🛡️ 安全风险 | 权限范围、数据暴露风险、是否有安全审计 | 高 |
| 🔄 维护活跃度 | 最近更新时间、Issue响应速度、贡献者数量 | 中 |
| 🎯 创新程度 | 是否解决新问题、技术方案是否新颖 | 低 |

**输出**：
```
MCP评估：
- 项目A：社区⭐5 + 功能⭐4 + 集成⭐3 + 安全⭐4 = 综合⭐4.0
- 项目B：社区⭐3 + 功能⭐5 + 集成⭐3 + 安全⭐5 = 综合⭐4.0
```
