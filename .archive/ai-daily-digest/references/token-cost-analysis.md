# Token成本分析方法

## 定价模型

| 类型 | 价格 | 说明 |
|------|------|------|
| 输入命中缓存 | 0.025元/1M tokens | 最便宜 |
| 输入未命中 | 3元/1M tokens | 120倍于命中 |
| 输出 | 6元/1M tokens | 最贵，240倍于命中 |
| 缓存命中率 | 97% | 默认设置 |

## 计算公式

```python
# 输入成本
input_cost_hit = input_tokens * cache_hit_rate * 0.025 / 1_000_000
input_cost_miss = input_tokens * (1 - cache_hit_rate) * 3 / 1_000_000
input_cost = input_cost_hit + input_cost_miss

# 输出成本
output_cost = output_tokens * 6 / 1_000_000

# 总成本
total_cost = input_cost + output_cost
```

## 成本构成分析

- **输入成本**：占比<1%，即使未命中缓存也远低于输出
- **输出成本**：占比>99%，是成本的主要来源
- **SKILL文件大小**：影响输入token，但对总成本影响有限

## 优化建议

1. **减少输出token**：精简报告内容，减少冗余描述
2. **提高缓存命中率**：保持提示词稳定，减少变化
3. **选择合适SKILL版本**：日常用折中版，调试用完整版

## 常见错误

- **单位错误**：0.025元/1M tokens ≠ 0.025元/1K tokens（差1000倍）
- **忽略输出成本**：输出token是成本的主要来源
- **忽略缓存命中率**：97%命中率下输入成本差异不大
