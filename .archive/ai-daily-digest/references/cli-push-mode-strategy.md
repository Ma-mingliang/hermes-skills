# CLI 交互模式推送策略

> 当用户在 CLI 聊天中要求日报时的推送方式

## 三种推送模式

### 1. Cron 模式（微信自动推送）
- Cron job 的 final response 自动推送到配置的微信通道
- 需要分段：`[1/N]...[N/N]` 格式
- 间隔：每段 60 秒（避免限流）

### 2. 交互模式（用户在当前聊天要求）
- 用户说"推送到这个聊天"、"推送到当前端口"、"在这个聊天中推送"
- 直接在回复中输出完整报告内容
- 不需要调用 send_message
- 不需要分段，一次性输出
- 报告长度无限制

### 3. CLI 模式（终端用户）
- 用户在 CLI 终端要求日报
- 直接在回复中输出完整报告
- 无需任何推送操作
- 格式：纯文本，无 markdown 渲染

## 执行流程（交互/CLI模式）

```
Step 1-5: 数据收集 → 验证 → 分类 → 生成报告（同标准流程）
Step 6: 质量检查（同标准流程）
Step 7: 直接在回复中输出完整报告内容
        - 不调用 send_message
        - 不写 push_payload.json
        - 不分段
```

## 关键区别

| 操作 | Cron模式 | 交互/CLI模式 |
|------|---------|-------------|
| 推送方式 | final response 自动推送 | 直接输出在回复中 |
| 是否分段 | 是（[1/N]格式） | 否（一次性输出） |
| 是否调用send_message | 否 | 否 |
| 间隔控制 | 60秒 | 无需 |
| 限流风险 | 有（需处理） | 无 |
