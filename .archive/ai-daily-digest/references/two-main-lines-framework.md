# AI发展双主线框架

## 核心逻辑

```
主线1：提高AI能力（让AI更强大）
    ↕ 相互促进、螺旋上升
主线2：让AI更好地解决人类问题（让AI更有用）
```

## 热词按两条主线分类

### 主线1：提高AI能力

| 阶段 | 热词 | 能力提升 |
|------|------|----------|
| 基础 | Transformer, Self-Attention | 架构创新 |
| 基础 | Pre-training, BERT, GPT | 知识学习 |
| 基础 | RLHF, Fine-tuning | 人类偏好 |
| 进阶 | Reasoning, CoT | 推理能力 |
| 进阶 | Multimodal | 多模态理解 |
| 进阶 | Long Context | 长文本处理 |
| 高级 | Quantization, Distillation | 模型压缩 |
| 高级 | Speculative Decoding | 推理加速 |
| 高级 | Edge AI | 本地部署 |

### 主线2：解决人类问题

| 阶段 | 热词 | 解决问题 |
|------|------|----------|
| 基础 | Prompt, Prompt Engineering | 如何与AI对话 |
| 基础 | RAG, Embedding, Vector DB | 让AI查资料 |
| 基础 | LangChain, Dify | 构建AI应用 |
| 进阶 | Agent, Function Calling | 让AI做事 |

| 高级 | Human-in-the-Loop | 人机协作 |
| 高级 | Vibe Coding | 降低开发门槛 |

## 三条交互模式

### 模式1：能力提升→问题暴露→新热词

```
主线1能力提升 → 用户期望提高 → 主线2问题暴露 → 需要新解决方案

示例：
- GPT-4能力强大 → 用户期望AI知道最新信息 → 知识截止问题暴露 → RAG技术
- Claude 3推理能力强 → 用户期望AI能"做事" → 无法执行动作问题暴露 → Agent架构
- DeepSeek R1推理能力强 → 用户期望AI能写代码 → 编程门槛问题暴露 → Vibe Coding
```

### 模式2：需求反馈→方向指引→新热词

```
主线2需求反馈 → 主线1研究方向 → 能力提升 → 更高需求

示例：
- 用户需要AI查资料 → 推动RAG研究 → RAG技术成熟 → 用户需要更精准检索
- 用户需要AI做事 → 推动Agent研究 → Agent架构成熟 → 用户需要更复杂任务
- 用户需要AI写代码 → 推动推理能力 → 推理能力提升 → 用户需要更复杂代码
```

### 模式3：交汇点产生新热词

```
主线1能力 + 主线2需求 → 交汇点 → 新热词

示例：
- 推理能力 + 编程需求 → Vibe Coding
- 工具调用 + 任务执行 → Agent
- 检索能力 + 知识需求 → RAG
- 对齐能力 + 安全需求 → Guardrails
```

## 完整时间线（含阻碍与解决方案）

### 第一阶段：AI发展历史（1950-2017）

| 时代 | 主线1阻碍 | 主线2阻碍 | 共同解决方案 |
|------|-----------|-----------|--------------|
| 1950s | 计算能力不足 | 无法处理复杂问题 | 硬件发展 |
| 1970s | 感知机局限 | 应用未兑现 | AI寒冬 |
| 1980s | 训练深层网络困难 | 知识获取瓶颈 | GPU+知识工程 |
| 1990s | 专家系统局限 | 维护成本高 | 统计学习 |
| 2010s | 深层网络训练困难 | 标注数据不足 | GPU+大数据 |
| 2017 | RNN无法并行 | 长文本理解困难 | Transformer |

### 第二阶段：大模型时代（2017-2026）

| 时代 | 主线1阻碍 | 主线2阻碍 | 共同解决方案 | 交互模式 |
|------|-----------|-----------|--------------|----------|
| 2018 | 标注数据不足 | 每个任务都要训练 | 预训练+微调 | 能力提升→新需求 |
| 2020 | 模型太大 | 需要通用能力 | 规模定律 | 能力提升→新可能 |
| 2022 | 输出不可控 | AI胡说八道 | RLHF | 需求反馈→对齐技术 |
| 2023 | 模型越大越贵 | AI知识截止 | RAG | 能力提升→问题暴露 |
| 2024 | 只会说不会做 | 无法执行动作 | Agent | 需求反馈→工具调用 |
| 2025 | 推理能力不足 | 编程门槛高 | Vibe Coding | 能力提升→门槛降低 |
| 2025 | 单Agent能力有限 | 复杂任务难完成 | Multi-Agent | 需求反馈→多Agent |
