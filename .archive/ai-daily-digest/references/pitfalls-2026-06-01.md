# ai-daily-digest Pitfalls (补充 2026-06-01)

## P60: 质量检查脚本格式陷阱
- **问题**：quality_check.py用精确字符串匹配检查报告格式
- **具体案例**：
  - "按Stars排名"被拒，必须用"按自主性"或"按成本"或"对比分析"
  - "第一类：减少token消耗"被拒，必须用"第一类"（无冒号无描述）
  - emoji编号检查："第二类："不能出现
- **修复**：FAIL时先读quality_check.py源码，找到check()函数的精确expect

## P61: WSL terminal不可用
- **问题**：Windows上terminal工具可能WSL报错
- **症状**：`WSL (xxx - Relay) ERROR: CreateProcessCommon:798: execvpe(/bin/bash) failed`
- **修复**：用execute_code + Python subprocess替代terminal

## P62: subagent无web访问能力
- **问题**：只有GitHub MCP tools的subagent无法访问任意URL
- **症状**：delegate_task的子任务无法执行curl/web_fetch
- **修复**：需要web请求时不用delegate_task，用父agent的execute_code直接做
