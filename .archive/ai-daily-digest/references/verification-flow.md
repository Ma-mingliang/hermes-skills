# AI日报验证流程

## 验证时机
生成报告后、推送前，必须运行验证循环。

## 验证Phase（11个、71项检查）

### Phase 1: 结构验证（8项）
检查报告是否包含所有8个板块。

### Phase 2: Agent生态验证（7项）
检查Agent定义、使用指南、高星全能/专精/组件表格。

### Phase 3: Skills市场验证（9项）
检查6类emoji、痛点/原理/案例/GitHub/前辈对比。

### Phase 4: 痛点描述验证（9项）
检查是否按9种问题类型框架描述：
- 能力缺失型、使用不便型、成本过高型、安全风险型
- 效率低下型、知识壁垒型、协作困难型、数据孤岛型、行业落地型

### Phase 5: 模型动态验证（4项）
检查官网监控、HN新闻、OpenRouter、对比网站。

### Phase 6: 行业应用验证（5项）
检查行业覆盖>=3个、表格、链接、来源。

### Phase 7: MCP动态验证（10项）
检查MCP定义、项目表、七大分类。

### Phase 8: 数据面板验证（6项）
检查今日/本周/历史区分、分类统计。

### Phase 9: 核心信号验证（2项）
检查信号>=3条、有热度标注。

### Phase 10: AI基础知识验证（7项）
检查目标/概念/重要性/误解/阅读/测验/主线。

### Phase 11: 格式验证（4项）
检查emoji编号、无重复标题、GitHub链接>=15。

## Fix Loop
如果验证失败：
1. 修复问题
2. 重新生成报告
3. 再次验证
4. 直到全部通过
