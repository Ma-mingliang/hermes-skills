# AI日报 V2 模板（用户确认格式 2026-05-29）

> 用户明确要求的输出格式，必须严格遵守。

## 完整模板结构

```markdown
# 🤖 AI日报 YYYY/MM/DD

---

## 一、Agent 智能体平台

> **定义**：可独立运行，接入API执行任务的平台

### 1️⃣ 全能型Agent

#### 📌 使用指南
- 简短核心技巧 + 链接，不展开
- 格式：**名称** / 热度 / 功能 / 核心技巧 / 链接 / 适用场景

#### 🆕 新出现
每个Agent格式：
- **名称** — 一句话定位
- **热度**：🔥 X HN points | 💬 Y comments
- **功能**：详细描述能做什么
- **核心优势**：列表形式
- **使用案例**：具体场景
- **GitHub**：链接 · **HN讨论**：链接
- **适用场景**：列表

**必须包含📊对比表+🔍拆分+📋归纳：**

📊 **对比分析**
| 维度 | Agent A | Agent B | Agent C |
|------|---------|---------|---------|
| **定位** | ... | ... | ... |
| **成本** | 免费/付费 | ... | ... |
| **自主性** | ⭐⭐⭐⭐⭐ | ... | ... |
| **易用性** | ⭐⭐⭐⭐ | ... | ... |
| **适用人群** | ... | ... | ... |

🔍 **拆分**：按XX维度：A > B > C
📋 **归纳**：场景→推荐Agent

#### ⭐ 高星Agent
格式同🆕新出现，但侧重Stars数和成熟度。
附📊对比表+🔍拆分+📋归纳。

#### 🧩 高星Agent组件
> **定义**：为Agent平台提供扩展能力的系统/工具

每个组件格式：
- **名称** ⭐Stars
- **解决问题**：原始Agent的什么不足
- **核心原理**：算法/架构层面
- **适用Agent**：支持哪些平台
- **GitHub**：链接

### 2️⃣ 专精型Agent
格式同全能型，含技术原理说明。

**⚠️ 如果当日没有新的专精agent，推荐5个高星的专精agent（含详细描述+前辈对比）！**

---

## 二、Skills 技能文档

> **定义**：.md规则文档，约束AI按规则执行，不能独立运行

### 第一类：减少token消耗
### 第二类：约束agent行为
### 第三类：增加功能
### 第四类：科研辅助
### 第五类：检测正常工作
### 第六类：补充类/其他

每个Skill格式：
- **名称** ⭐Stars
- **功能**：详细描述
- **原理**：算法/技术原理
- **效果**：量化效果
- **GitHub**：链接
- **适用场景**：列表

---

## 三、模型动态

格式：
**🔴 模型名称 版本号**
- 🔥 X HN points | 💬 Y comments
- 内容描述
- 影响分析
- 链接：[来源](url) · [HN讨论](url)

**⚠️ 模型排名必须精确到小版本：opus4.8、gpt5.5、deepseekv4pro等**

---

## 四、行业应用

表格格式：
| 行业 | 事件 | 来源 |
|------|------|------|
| AI基础设施 | 事件描述 | [来源](url) · [HN](url) |

**⚠️ 每个事件必须附可点击链接！**

---

## 五、今日数据面板

| 类别 | 数量 | 重要度 |
|------|------|--------|
| 🤖 全能型Agent | X个 | 🔴极高 |
| 🤖 专精型Agent | X个 | 🟡重要 |
| 🛠️ Skills | X个 | 🔴高 |
| 📊 模型动态 | X条 | 🔴极高 |
| 🏭 行业应用 | X条 | 🟡重要 |

---

## 六、核心信号

5条核心信号，带emoji编号：
1. **🔥 信号一**
2. **💰 信号二**
3. **📜 信号三**
4. **🧠 信号四**
5. **✍️ 信号五**

---

⏰ 生成时间 | 📡 数据来源 | ✅ 分类标准
```

## 关键格式规则

1. **用emoji编号**：📌/🆕/⭐/🧩，不用"第二类/第三类"
2. **对比表维度**：定位/成本/自主性/易用性/适用人群（五星评分）
3. **拆分**：按不同维度排序（如"按自主性：A > B > C"）
4. **归纳**：按场景推荐（如"企业自动化→n8n/Dify"）
5. **AI基础知识板块**：用户说"暂时不出现"时省略
6. **模型版本**：精确到小版本号
7. **链接**：每个内容项必须附GitHub链接
8. **前辈对比**：新Agent/Skills必须找前辈对比分析
9. **详细描述**：所有内容必须详细描述功能和应用领域
