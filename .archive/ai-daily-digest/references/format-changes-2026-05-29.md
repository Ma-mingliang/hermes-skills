# 格式变更记录（2026-05-29）

## 用户明确纠正的格式要求

### 1. emoji编号替代文字编号
- ❌ 错误：「第二类：新出现的全能型Agent」
- ✅ 正确：「🆕 新出现」

### 2. 对比表必须包含五星评分
- 列：定位/成本/自主性(⭐评分)/易用性(⭐评分)/适用人群
- 示例：自主性 ⭐⭐⭐⭐⭐

### 3. 拆分格式
- 按不同维度给出排序结论
- 示例：「按自主性：DeerFlow ≈ Hermes > n8n ≈ Dify > ECC ≈ Langflow」

### 4. 归纳格式
- 按场景给出推荐
- 示例：「企业自动化→n8n/Dify，通用助手→Hermes，快速原型→Langflow」

### 5. Agent组件是第三分类
- Agent = 可独立运行平台
- Skills = .md规则文档
- Agent组件 = 跨平台优化系统（如ECC）
- 三者绝不能混淆

### 6. 新Agent/Skills必须找前辈对比
- 找到功能相似的先行者
- 对比差异/相同
- 说明后者解决前者什么问题
- 说明解决方案原理

### 7. Skills需对比高星Skills
- 找star更高、数量较少的同类skills
- 可加入过去高star的skills做对比

### 8. 专精Agent保底5个
- 当日没有新的专精agent时
- 推荐5个高星的专精agent
- 含详细描述+前辈对比

### 9. 模型排名精确到小版本
- ❌ 错误：Claude 4
- ✅ 正确：Claude Opus 4.8

### 10. 所有内容必须详细描述
- 不能只写一句话
- 必须包含：应用领域、核心功能、技术原理、使用案例
