# WeChat 分段推送策略

## 问题描述

微信 iLink 有账户级限流（ret=-2），连续发送 4+ 条消息会触发。

## 推送策略

### 长报告分段推送

1. **分段大小**：每段 2000-2500 字符
2. **间隔时间**：每段间隔 **2 分钟**（120 秒）
3. **限流后等待**：如果触发限流，等待 **5 分钟**后重试
4. **持续限流**：如果 5 分钟后仍限流，等待 **2-6 小时**自动解除

### 分段规则

- 按 `##` 或 `###` 标题自然分段
- 每段开头标注 `Part X/Y`
- 避免在段落中间断开

### 限流处理

```python
import time

# 发送每段后等待 2 分钟
time.sleep(120)

# 如果触发限流，等待 5 分钟
time.sleep(300)

# 如果仍限流，建议用户等待 2-6 小时
```

### 替代方案

如果限流持续：
1. 设 cron job 定时发送
2. 直接读取报告文件
3. 分多天发送（每天发一部分）

## 最佳实践

1. **合并消息**：尽量合并为 2-3 条长消息（3000-4000 字符）
2. **避免频繁发送**：减少消息数量
3. **使用 cron job**：定时任务使用 cron job 的 final response 自动推送
4. **监控限流**：检查 `last_delivery_error` 是否包含 `rate limited: ret=-2`

## 故障排查

如果 cron job 显示 `rate limited: ret=-2`：

1. 清理 stale context_token
2. 使用全局 `Hermes_Gateway.cmd` 重启（不能用项目 hermes.bat）
3. 等待 10 秒，检查 gateway_state.json 确认 weixin: connected
