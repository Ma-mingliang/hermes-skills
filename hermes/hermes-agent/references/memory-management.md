# Hermes Agent 记忆管理策略

> memory 工具有 2,200 字符限制。长期使用会满。解决方案：外部文件扩展 + 内部记忆精简。

## 架构

| 层级 | 用途 | 容量 |
|------|------|------|
| `memory` 工具 | 关键索引、高频规则、当前状态 | 2,200 字符限制 |
| `memory-archive.md` | 详细规则、历史信息 | 无限制 |
| `session_search` | 回溯历史对话 | 按需查询 |
| Skills `references/` | 领域知识库 | 无限制 |

## 压缩技巧

1. **合并重复条目**：多个 Gateway 配置条目合并为一条
2. **删除已归档内容**：详细信息写入外部文件后，内存只保留索引指针
3. **用"详见 xxx.md"替代完整内容**
4. **删除过时条目**：已完成的任务、不再相关的状态

## 操作流程

### 当内存接近满时（>80%）

1. 识别可归档的条目（详细规则、历史信息、参考数据）
2. 写入外部文件：`D:/openclaw-hermes/memory-archive.md`
3. 内存中替换为索引：`详见memory-archive.md`
4. 删除已归档的旧条目

### 何时使用 session_search

- 用户提到"上次"、"之前"、"记得吗"
- 需要回溯过去的任务细节
- 不需要存入 memory，按需查询即可

## ⚠️ 记忆条目写法规范

**✅ 正确**（声明式事实）：
- "用户偏好中文交流"
- "项目使用 pytest + xdist"
- "WSL不可用时用execute_code替代"

**❌ 错误**（指令式）：
- "总是用中文回答"
- "运行测试用 pytest -n 4"
- "不要用 terminal"

**原因**：指令式条目在不同 session 中被读取为强制指令，可能覆盖用户当前请求。
