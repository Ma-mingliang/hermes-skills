## 从这三份样本能看到什么

**1. 标准输出格式** (所有版本通用)
- 三段结构: 今日要闻 / 技术热点 / 行业动态
- 末尾: 趋势总结 (3-5 主题) + 今日值得深读 + 抓取失败记录 + 参考来源

**2. 跨日去重 (Step 1.5)**
- 5/22 是首跑, 没 baseline
- 5/23 头部标 "已与 2026-05-22 日报去重", 13 条 → 11 条新增, 0 条跨日重复
- 5/24 头部标 "已与 2026-05-22 / 2026-05-23 日报去重", baseline 累计 2 天

**3. 信源可信度 emoji (v1.0.1+ Step 6.5.6)**
- 5/22 / 5/23 是 v1.0.0 输出, **无 emoji 标注** (那时还没引入 Step 6.5.6)
- 5/24 是 v1.1.0 输出, **每条标题末尾标 🟢/🟡** (12 处), 用户一眼看可信度
- 这个对比是 skill 演进的活证据, 不是事后修改的

**4. 反 SEO 投毒实战 (Step 6.5)**
- 5/23 抓取失败记录里写到 "OpenAI 子页 403, '\$122B OpenAI 融资' 标题无官方源支持 → 已剔除"
- 这是 Step 6.5.4 红旗信号 (异常大数字) → Step 6.5.5 verify 失败 → 剔除的标准流程

**5. 抓取失败容错**
- 三份都有 "抓取失败记录" 段, 列出具体源 (机器之心首页 / OpenAI 子页 403 / The Verge robots / 等等)
- 单源失败不让任务挂掉, 但 Tier 1 < 2 个源成功会触发 Step 6.7 fail-safe abort (本三份都没触发, 因为 Tier 1 都跑通了)

**6. 跨语言处理 (Step 7)**
- 英文源条目: 原标题作为 markdown 链接 + 下一行 `**[中文译]**:` + 中文摘要
- 中文源条目: 标题保持中文, 摘要中文, 不加 [中文译]

## 自己跑一份

装好 skill 后 (见上一级 [README.md](../README.md) 的 Install 段), 在你的 agent 里说:

```
生成今日 AI 日报
```

输出会落到 `~/Desktop/ai-news/YYYY-MM-DD.md`。

如果今天已经跑过, 同日重跑行为取决于 runtime (Step 1.6):
- 交互式 agent (Claude Code / Cursor) 会问你 A 更新 / B 副本 / C 跳过
- 非交互 agent (Codex / cron / API) 默认走 A 更新累积

也可以显式指定: `生成今日 AI 日报 mode=snapshot`
