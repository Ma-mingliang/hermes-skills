# 流程思维 Skill

拆分事件，将任务分成细致的环节，在执行任务前进行分析和规划。

## 🎯 核心定位

**用途**：在执行任务前进行流程分析和拆分  
**目标**：将复杂任务分解为可执行的环节  
**价值**：提高任务执行效率和质量

## 🔄 核心流程

```
接收任务 → 分析任务 → 拆分环节 → 定义流程 → 执行任务 → 复盘优化
```

## 🔧 核心功能

### 1. 任务拆分
将复杂任务拆分成多个环节

**示例：学习流程**
```
预习 → 听课 → 练习 → 复习 → 考试 → 复盘 → 单元测验
```

**示例：开发流程**
```
需求分析 → 设计 → 编码 → 测试 → 部署 → 监控 → 优化
```

**示例：写作流程**
```
选题 → 大纲 → 初稿 → 修改 → 校对 → 发布 → 反馈
```

### 2. 流程定义
定义每个环节的具体内容

**示例：预习环节**
- 阅读教材
- 标记疑问
- 准备问题

### 3. 流程执行
按照流程执行任务

**执行方式**：
- 按顺序执行每个环节
- 记录执行过程
- 标记完成状态

### 4. 流程优化
根据执行结果优化流程

**优化方式**：
- 分析执行结果
- 识别问题环节
- 调整流程顺序
- 改进环节内容

## 🔍 执行验证循环（2026-05-31新增）

**问题**：有流程但不按流程执行，导致任务质量低
**解决方案**：执行验证循环（test-verify-fix loop）

**核心思想**：
- 每个环节执行后必须验证
- 验证通过才进入下一个环节
- 验证失败必须修复并重新验证

**执行验证循环流程**：
```
Step N: 执行
  ↓
Step N.5: 验证（对照清单逐项检查）
  ↓
通过？ → 是 → 进入Step N+1
  ↓
否 → 修复 → 回到Step N.5重新验证
```

**验证清单示例**：

每个Step完成后，必须对照清单验证：

**数据收集验证清单**：
- [ ] GitHub API: 收集了X个项目，今日新增Y个
- [ ] HN API: 收集了X个帖子，今日新增Y个
- [ ] 36氪RSS: 收集了X条新闻，今日新增Y条
- [ ] 模型官网: 访问成功X/8
- [ ] 模型排名: 访问成功X/2
- [ ] 所有数据是否保存到文件
- [ ] 执行记录是否保存

**分类验证清单**：
- [ ] 是否执行了Step 1-6分类流程
- [ ] 是否有MCP分类
- [ ] 是否有Agent细分（全能/专精）
- [ ] 是否有Skills细分（6类）
- [ ] 是否执行了交叉验证
- [ ] 分类记录是否保存

**报告生成验证清单**：
- [ ] Agent报告是否包含速览表
- [ ] 每个Agent是否包含痛点+功能+原理+前辈对比
- [ ] Skills报告是否包含详细分析
- [ ] 每个Skill是否包含痛点+原理+案例+前辈对比
- [ ] 模型动态是否提供对比网站链接
- [ ] 行业应用是否覆盖≥3个行业
- [ ] MCP动态是否包含具体项目
- [ ] 报告记录是否保存

**关键价值**：
1. 确保每个环节都执行到位
2. 避免遗漏关键步骤
3. 提高任务执行质量
4. 形成可验证的执行记录

**参考项目**：
- [OpenAgentsControl](https://github.com/darrenhinde/OpenAgentsControl) (⭐4184): plan-first + approval-based execution
- [prax-agent](https://github.com/ChanningLua/prax-agent) (⭐295): test-verify-fix loops
- [kweaver-core](https://github.com/kweaver-ai/kweaver-core) (⭐808): verifiable feedback loops

## 🔍 工作流调试模式（2026-05-30新增）

当工作流执行出错时，使用思维追踪法找根因：

### 调试流程
```
1. 列出每个环节的实际执行结果
2. 对比预期结果，标记差异
3. 追踪差异的根因（不是表面现象）
4. 修复根因，不是修补表面
```

### 案例：AI日报工作流调试
**现象**：推荐的agent不是当日的

**环节追踪**：
| 环节 | 预期 | 实际 | 差异 |
|------|------|------|------|
| 时间戳 | 当日0点 | 24小时前 | ❌ |
| GitHub查询 | 今日新增 | 近7天 | ❌ |
| 数据验证 | 验证时间戳 | 跳过 | ❌ |

**根因分析**：
- 表面：GitHub没有今日数据
- 真正：时间戳计算错误（`time.time() - 86400` ≠ 当日0点）
- 修复：用`today.replace(hour=0, minute=0, second=0).timestamp()`

### 调试原则
1. **不要停在表面现象**："没有数据"是现象，不是原因
2. **逐环节验证**：每个环节都要检查输入/输出是否符合预期
3. **找最早出错的环节**：后面的问题往往是前面问题的连锁反应
4. **修复根因**：修时间戳，不是"放宽搜索条件"

## 🔧 API参考

### 核心函数

#### `decompose_task(task_description, task_type=None) -> Dict`
拆分任务为多个环节

**参数**：
- `task_description`: 任务描述
- `task_type`: 任务类型（learning/development/writing/custom）

**返回**：
```python
{
    "task": "任务描述",
    "type": "任务类型",
    "steps": [
        {
            "step": 1,
            "name": "环节名称",
            "description": "环节描述",
            "tasks": ["具体任务1", "具体任务2"],
            "estimated_time": "预计时间",
            "dependencies": []
        }
    ],
    "total_steps": 7,
    "estimated_total_time": "预计总时间"
}
```

#### `get_process_template(process_type) -> Dict`
获取预定义流程模板

**参数**：
- `process_type`: 流程类型（learning/development/writing）

**返回**：流程模板

#### `create_custom_process(name, steps) -> Dict`
创建自定义流程

**参数**：
- `name`: 流程名称
- `steps`: 环节列表

**返回**：自定义流程

#### `format_process(process, format="text") -> str`
格式化流程

**参数**：
- `process`: 流程定义
- `format`: 输出格式（text/json/markdown）

**返回**：格式化的流程
